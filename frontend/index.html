SSL-Zertifikat installieren

Damit Browser diese Verbindung als sicher einstufen, muss das Zertifikat importiert werden.

Zertifikat herunterladen (jarvis.cer)
  1. Lade die Datei jarvis.cer herunter.
  2. Rechtsklick auf die Datei → "Zertifikat installieren".
  3. Wähle "Lokaler Computer" (nicht "Aktueller Benutzer").
  4. Wähle "Alle Zertifikate in folgendem Speicher speichern".
  5. Klicke auf "Durchsuchen" und wähle "Vertrauenswürdige Stammzertifizierungsstellen".
  6. Bestätigen und Fertigstellen.
  7. Browser komplett neu starten (alle Tabs schließen).
# jarvis.cer liegt im DER-Format vor -> nach PEM konvertieren
openssl x509 -inform DER -in jarvis.cer -out jarvis-ca.crt

# 1) System-CA-Store (fuer curl/wget/git etc.) – .crt-Endung ist Pflicht
sudo cp jarvis-ca.crt /usr/local/share/ca-certificates/jarvis-ca.crt
sudo update-ca-certificates

# 2) Chrome/Chromium nutzen einen EIGENEN Store (NSS) – Pflicht fuer Autofill
sudo apt install -y libnss3-tools   # falls 'certutil' fehlt
mkdir -p $HOME/.pki/nssdb
certutil -d sql:$HOME/.pki/nssdb -A -t "C,," -n "Jarvis CA" -i jarvis-ca.crt

# danach Chrome komplett neu starten

Wichtig: Chrome/Chromium unter Linux vertrauen NUR dem NSS-Store (Schritt 2) – der System-Store allein reicht fuer das Passwort-Autofill nicht. Firefox: siehe Tab "Browser-Import".

Falls die System-Installation nicht greift (z.B. Firefox):

  1. Öffne die Einstellungen deines Browsers.
  2. Suche nach "Zertifikate" oder "Zertifikate verwalten".
  3. Klicke auf "Importieren..." im Reiter "Zertifizierungsstellen" (Authorities).
  4. Wähle die Datei jarvis.cer aus.
  5. Setze den Haken bei "Dieser CA vertrauen, um Websites zu identifizieren".

🔌 Support-API – REST-Zugriffe

Der Support-Assistent ist auch ohne Web-Oberfläche per REST erreichbar – z. B. aus Ticketsystemen, Skripten oder anderen Anwendungen. Basis-URL ist dein Jarvis-Host (https://DEIN-JARVIS-HOST). Alle Bodies sind JSON (Content-Type: application/json).

Authentifizierung

  • Angemeldeter Benutzer (Browser/Sitzung): Header Authorization: Bearer <token>
  • Externe Anwendung: Header X-API-Key: <API-Key> – API-Keys werden in den Einstellungen verwaltet. Nur /query und /summarize akzeptieren API-Keys; die übrigen Endpunkte erfordern ein Benutzer-Token.

Endpunkte

MethodePfad ZweckAuth
POST/api/support/querySuche über RAG + Jira + Confluence, optional KI-ZusammenfassungToken oder API-Key
POST/api/support/summarizeKI-Zusammenfassung eines einzelnen Jira-TicketsToken oder API-Key
GET/api/support/statusAktive Quellen + Maxima (Zeilen, Ticketanzahl)Token
GET/POST/api/support/instructionsPersönliche Anweisungen lesen/speichern (Markdown)Token
GET/DELETE/api/support/historySuchverlauf des Benutzers lesen/löschenToken

Beispiel 1 – Suche mit KI-Zusammenfassung

Durchsucht Wissensdatenbank und (nur offene) Jira-Tickets, holt bis zu 10 Tickets und lässt eine KI-Gesamtzusammenfassung erstellen.

curl -sk https://DEIN-JARVIS-HOST/api/support/query \
  -H "X-API-Key: DEIN_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
        "text": "Drucker im 2. OG offline",
        "rag": true,
        "jira": true,
        "confluence": false,
        "ai": true,
        "open_only": true,
        "jira_limit": 10,
        "summary_lines": 5,
        "lang": "de"
      }'

Antwort (gekürzt):

{
  "ok": true,
  "query": "Drucker im 2. OG offline",
  "ai_summary": "Mögliche Ursache ist ein Netzwerk-Timeout des Druckers …",
  "jira_total": 7,
  "blocks": [
    { "source": "JIRA",   "key": "SUP-1234", "title": "SUP-1234",
      "score": 92, "summary": "Drucker reagiert nicht …" },
    { "source": "WISSEN", "title": "Drucker-Reset Anleitung",
      "score": 80, "summary": "Schritt 1: …" }
  ]
}

blocks[].source ist WISSEN, JIRA oder CONFLUENCE; score ist die Relevanz in %. Felder: text (Pflicht), rag/jira/confluence/ai/open_only (bool), jira_limit (1…Maximum), summary_lines, lang (de/en).

Beispiel 2 – Einzelnes Ticket zusammenfassen

Lädt ein konkretes Jira-Ticket (Beschreibung + Kommentare) und fasst es in wenigen Sätzen zusammen.

curl -sk https://DEIN-JARVIS-HOST/api/support/summarize \
  -H "X-API-Key: DEIN_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "source": "JIRA", "key": "SUP-1234", "lang": "de" }'

Antwort:

{
  "ok": true,
  "key": "SUP-1234",
  "summary": "Das Ticket beschreibt einen Druckerausfall im 2. OG …"
}

Hinweise: -k akzeptiert das selbstsignierte Zertifikat (für Tests). Fehler liefern { "ok": false, "error": "…" } mit Status 401 (Auth), 403 (Skill inaktiv) oder 400 (ungültige Eingabe).
